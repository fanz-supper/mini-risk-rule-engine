# Mini Risk Rule Engine

# 项目背景 / 为什么要做这个轮子

在实际业务中，风控逻辑通常非常复杂且频繁变更，例如：

- 首单用户下单金额异常偏大；
- 同一设备短时间内频繁注册 / 登录；
- 多个账号共用同一张银行卡或同一台设备；
- IP、设备指纹命中黑名单 / 灰名单等。

很多团队会把这些判断写成大量的 `if...else...` 散落在各个服务里，带来几个典型问题：

1. **规则难以维护**：
    规则逻辑分散在代码中，不同业务线有自己的风控判断，很难整体查看 & 管理。
2. **策略调整成本高**：
    只要修改一个条件（比如金额阈值、时间窗口），就需要改代码、发版、回归测试。
3. **缺乏可观测性**：
    很难回答「这一次拦截/放行是因为触发了哪几条规则」。

**Mini Risk Rule Engine** 的目标就是解决这些痛点：

- 将风控逻辑从硬编码转为**配置化规则**；
- 提供一个通用的**规则解析与执行引擎**；
- 支持规则的**分组、优先级、组合策略**；
- 提供简单的**命中日志 & 决策解释**，便于排查与调优。

这是一个适合学习和实践的中间件级轮子项目，而不是简单的 CRUD Demo。
